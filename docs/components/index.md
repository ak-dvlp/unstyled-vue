# Общий обзор компонентов

В целом компоненты библиотеки не содержат стилей, хотя некоторые стили всё же используются, например, для обеспечения желаемого поведения при работе с программами экранного доступа.

Компоненты библиотеки поддерживают стилизацию через параметр `classes` объектного типа. Ключи этого объекта указаны в таблице входных параметров, а их расположение в структуре шаблона компонента можно найти в выделенных строках раздела `Шаблон`. Допустимо использование обычных `CSS-классов`, `классов Tailwind` или их комбинации.

::: warning Предупреждение
Примеры компонентов в разделе `Примеры` используют стандартные служебные классы `Tailwind`, а также пользовательские служебные классы, определённые через директиву встроенной темы `@theme inline`. Поэтому прямое копирование и вставка стилей из примеров может не дать желаемого результата. <br /> Пользовательские классы можно легко узнать по приставке `my`. Например: `text-my-label`.
:::

Обязательные параметры компонентов отмечены красной звёздочкой <span style="color: var(--vp-custom-block-danger-text); font-size: 1.25rem;">\*</span>. <br />
У многих обязательных и у всех необязательных параметров заданы значения по умолчанию. Это помогает обеспечить типобезопасность, а также стабильную работу компонентов в разных средах сборки и в тех случаях, когда динамические данные временно отсутствуют.

Раздел `Комментарии к устройству компонента`, который встречается на некоторых страницах, не имеет прямого отношения к документации и служит лишь для моих личных заметок.
