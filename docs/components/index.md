# Общий обзор компонентов

В целом компоненты библиотеки не содержат стилей, хотя некоторые стили всё же используются, например, для обеспечения желаемого поведения при работе с программами экранного доступа.

Компоненты библиотеки поддерживают стилизацию через параметр `classes` объектного типа. Ключи этого объекта указаны в выделенных строках раздела `Шаблон` на странице каждого компонента. Допустимо использование обычных `CSS-классов`, `классов Tailwind` или их комбинации.

Обязательные параметры компонентов отмечены красной звёздочкой <span style="color: var(--vp-custom-block-danger-text); font-size: 1.25rem;">\*</span>. <br />
У многих обязательных и у всех необязательных параметров заданы значения по умолчанию. Это помогает обеспечить типобезопасность, стабильную работу компонента в разных средах сборки, а также в тех случаях, когда динамические данные временно отсутствуют.

Раздел `Комментарии к устройству компонента`, который встречается на некоторых страницах, не имеет прямого отношения к документации и служит лишь для моих личных заметок.
